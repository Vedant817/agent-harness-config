# Web App Criteria
(extends `_shared-core.md`)

- **Functionality:** Can a user actually complete the core flows end to end through the running UI — not just see the elements that imply the flow exists? Click-through with Playwright, don't infer from code.
- **Design coherence:** Does the UI read as one deliberate visual identity (colors, type, spacing, layout) rather than a collection of unrelated default components?
- **Originality:** Penalize generic "AI slop" patterns — unmodified stock components, the default purple-gradient-on-white-card look, template layouts with no adaptation to this product's purpose.
- **Workflow clarity:** Does the UI itself guide the user toward the correct sequence of actions, or does it require out-of-band knowledge (reading the spec, guessing) to use correctly?
